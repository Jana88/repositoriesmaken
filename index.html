Repositories & webpagina genereren

Giddy hub, 27 april 2014

Wat is een 'repositorie'? Dat is niet zo makkelijk uit te leggen. Je 'organisatie', die al aan bod is gekomen in een eerdere blog, kan vergeleken worden met de computer die je gebruikt. Je kan dan een 'repositorie' een beetje vergelijken met een map waar jouw bestanden in opgeslagen zijn. Uiteindelijk gaat al het werk dat je maakt in deze 'repositorie' komen te staan. Als het momenteel nog niet duidelijk is, vrees dan niet, dat zal het na enige oefening wel worden.

Nu de volgende stap, hoe maak je een 'repositorie'? In veel manieren is dit vergelijkbaar met de handelingen die je maakte als je een 'organisatie' hebt aangemaakt. Je kan dit op verschillende manieren doen. Ten eerste kun je weer klikken op het plusteken naast je accountnaam (rechter bovenhoek van je scherm) en kies je voor een 'new repository'.

Als je echter werkt vanuit je eigen account, en dus geen 'organisatie' hebt, krijg je normaal gezien een kader waar je toekomstige 'repositories' in komen te staan (nu leeg). Je kan daar op de groene knop genaamd 'new repository' klikken.

Vervolgens vul je de naam in die jouw 'repositorie' moet krijgen. Zorg er wel voor dat deze naam aan elkaar is geschreven of onderbroken is aan de hand van een '-'. Bovendien kun je geen andere leestekens gebruiken. Doe je dit toch, dan worden deze leestekens automatisch omgezet in de correcte vorm. In geval de naam reeds bestaat of als hij niet correct genoeg is, dan zal github je dit ook vertellen. Als er een groene vink achter verschijnt, kun je ervan uitgaan dat het in orde is.

Maak je niet te veel zorgen over de andere zaken die je kunt aanduiden. Vele tutorials zullen zeggen dat je deze moet aanduiden, maar het kan ook gemakkelijk zonder. Eens je de naam hebt ingevuld, klik je vervolgens op de groen knop 'create repository'. Het kan zijn dat je twee keer moet klikken.

Als alles goed is gegaan, krijg je nu het volgende scherm:

Maak je niet teveel zorgen over wat dit allemaal betekent. Dit is voornamelijk bedoeld voor zij die het programma 'Git' gebruiken om hun documenten, afbeeldingen, enzovoort op Github te zetten. Voor je nu eigenlijke blogs ed op je eigen website (pagina) kunt zetten, moet je nog wat stappen doorlopen. Aan de linkerkant kun je een aantal mogelijke tools terugvinden, waaronder 'settings'.Je hebt deze nodig elke keer je aanpassingen wilt maken en doorvoeren aan je webpagina. Dus, klik op 'settings'

Vervolgens kun je de naam van je 'repositorie' veranderen, je kan je 'repositorie' deleten, ... Let op, de naam veranderen van je 'repositorie' wilt niet zeggen dat je de naam verandert van je eigenlijke pagina. De naam die je jouw 'repositorie' hebt gegeven is wat er zal verschijnen in het webadres. De eigenlijke naam van je pagina veranderen komt later aan bod.

Klik nu op de 'automatic page generator'.

Je kan op dit moment jouw pagina een naam geven, dat wilt zeggen een naam die vanboven op deze webpagina zal verschijnen. Bovendien kun je bij 'tagline' een subnaam ingeven. Dit is wat er onder jouw titel zal verschijnen. Je kan ook de eigenlijke inhoud invoegen in het 'body' gedeelte. Let wel op dat wat je hier invult niet gewoon tekst is, maar een combinatie van tekst en html codes. Het gebruik van deze html codes komt later nog aan bod.

Klik vervolgens op de groene knop genaamd 'Continue to LayOut'.

Je kan nu je eigen tekst bewonderen in een preview van hoe het eruit gaat zien. Wil je een andere lay-out dan diegene die Github standaard aanbied, kan je dit in de bovenste balk kiezen. Let op, je kan nog verder kiezen dan de eerste vijf die er gegeven staat.

Eens je favoriete lay-out gekozen, en als alles klopt wat je ziet, kun je op de 'publish' knop drukken. Heb je nog een fout gezien, druk dan op de knop 'edit'.

je komt automatisch uit op de werkplek van je net aangemaakte 'repositorie'. Aan de bovenkant van je scherm verschijnt een blauwe balk met daarin je nieuwe webadres. Het kan wel zijn dat de verandering die je maakt pas na tien minuten zijn doorgevoerd op de eigenlijke website.

Zorg er wel altijd voor dat je werkt in de 'organisatie' of 'account' waar je jouw werk wilt zetten!
